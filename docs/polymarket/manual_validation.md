# Manual Validation Log — Phase 1 鯨魚情報驗證

> 此檔是 Phase 1 → Phase 2 的關鍵閘門。
> 架構規定：必須 2 個月內手動跟單記錄為「正期望值」，才能進 Phase 2 紙上自動化。
>
> 對應 docs/polymarket/architecture.md 第一章原則 1（情報先於交易）。

## 使用說明

每次 Telegram 收到 `[POLY-A/B/C]` 鯨魚推播時，**人類做決策**：

1. 若「我認為這個訊號值得跟單」→ 在下方表格記錄一筆
2. 不論是否真實下單（這階段都不該真實下單），記下「假設我跟了會如何」
3. 推播後 1-2 週（市場結算後）回填「實際結果」與「假設 PnL」

每月底彙總：
- 該月正確判斷數 / 總數
- 累積假設 PnL
- 三層（A/B/C）的個別命中率

## 驗收標準

進入 Phase 2 的條件：
- [ ] 至少 50 筆完整紀錄（含結算結果）
- [ ] 累積假設 PnL > 0（任何金額皆可，重點是方向）
- [ ] 三層中至少一層命中率 ≥ 60%
- [ ] 觀察時間 ≥ 60 天

任一條件未滿足，**不得**啟動 Phase 2。
